How It Works

The Verification Flow

From submission to publication. Every step verified, every action recorded.

1

Admin Submits Announcement

An administrator attempts to post a link-containing message to a protected channel. Validify intercepts and holds the content.

2

Approvers Notified

Designated approvers receive the message preview and a unique cryptographic verification code. Each code is single-use and time-limited.

3

Threshold Reached

Once the configured number of approvers submit their codes (e.g., 3 of 5), the verification threshold is satisfied. If the window expires, the message is rejected.

4

Published & Attested

The verified message is published with a Validify badge. Approval signatures are recorded on-chain, creating an immutable audit trail.

FAQ

Frequently Asked Questions

Will Validify ever ask for my seed phrase?
NEVER. Validify only uses wallet signature verification. We never ask for, store, or have access to your private keys or seed phrases. If anyone claiming to be from Validify asks for this information, it's a scam.
How does multi-signature verification work?
When an admin submits a link-containing announcement, Validify intercepts and holds it. Designated approvers receive unique cryptographic codes. Each approver has a time-limited window to verify. Only when your threshold (e.g., 3 of 5 admins) is met does the message publish. All approvals are recorded on-chain.
What platforms does Validify support?
Discord integration launches with the protocol in Q1 2026. Telegram expansion follows in Q2 2026. Our public API and SDK (Q3 2026) will enable integration with any platform. We're building infrastructure, not just a single-platform tool.
Will there be a token?
Token utility is planned for 2027, after the core protocol proves value. We're focused on building real security infrastructure first. Token design will be informed by actual usage patterns, not speculation. Details will be published in the 2027 roadmap.
When does Validify launch?
Protocol launches Q1 2026 with Discord integration, multi-sig verification, and on-chain attestations. We're currently onboarding founding communities for early access. Apply now to be among the first protected.
How is Validify different from existing security bots?
Existing solutions detect threats after publication‚Äîwhen damage is already done. Validify verifies before publication. We're not building a better blocklist. We're building a verification layer that makes unauthorized announcements impossible.
What if Validify itself gets compromised?
Validify cannot post on your behalf‚Äîit can only prevent unauthorized posts. Even if our systems were compromised, attackers couldn't use Validify to publish malicious content. The worst case scenario is verification temporarily going offline, not active exploitation. All smart contracts undergo third-party security audits before mainnet deployment.
What data does Validify collect?
We collect only what's necessary for security: message hashes (not content), approval signatures, and threat indicators. On-chain attestations use privacy-preserving hashed identifiers‚Äîyour community can't be enumerated from public blockchain data. We never store message content, user DMs, or personal information. Full privacy policy available at launch.
What if approvers aren't available when I need to post?
This is the security model working as intended. Legitimate announcements‚Äîmints, partnerships, important updates‚Äîare planned in advance. "I need to post this link RIGHT NOW without verification" is exactly what a compromised account would say. If your team can't coordinate approval for a critical announcement, that's a signal to pause, not bypass security.
How much will Validify cost?
Pricing will be announced closer to launch. Founding communities in our early access program receive extended free protection periods. We're designing pricing tiers that scale with community size and protection needs‚Äîserious security shouldn't be limited to projects with massive budgets.
