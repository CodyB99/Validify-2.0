How It Works

The Verification Flow

From submission to publication. Every step verified, every action recorded.

1

Admin Submits Announcement

An administrator attempts to post a link-containing message to a protected channel. Validify intercepts and holds the content.

2

Approvers Notified

Designated approvers receive the message preview and a unique cryptographic verification code. Each code is single-use and time-limited.

3

Threshold Reached

Once the configured number of approvers submit their codes (e.g., 3 of 5), the verification threshold is satisfied. If the window expires, the message is rejected.

4

Published & Attested

The verified message is published with a Validify badge. Approval signatures are recorded on-chain, creating an immutable audit trail.

5

User Verifies in Browser

When a community member clicks the verified link, the Validify browser extension automatically checks: Does this link match an attested announcement? Is the contract address verified? Are there any threat indicators? Users see a clear trust badge before connecting their wallet.

FAQ

Frequently Asked Questions

Will Validify ever ask for my seed phrase?
NEVER. Validify only uses wallet signature verification. We never ask for, store, or have access to your private keys or seed phrases. If anyone claiming to be from Validify asks for this information, it's a scam.
How does multi-signature verification work?
When an admin submits a link-containing announcement, Validify intercepts and holds it. Designated approvers receive unique cryptographic codes. Each approver has a time-limited window to verify. Only when your threshold (e.g., 3 of 5 admins) is met does the message publish. All approvals are recorded on-chain.
How is Validify different from existing security bots?

Existing security bots protect Discord servers. They detect threats after posting, and their protection ends when users click a link and leave Discord.

Validify protects the entire journey:

  • Server-side: Multi-signature verification BEFORE any link is posted
  • Browser-side: Extension verifies links at point of wallet connection
  • Identity: Hardware wallet binding proves admin legitimacy
  • Audit trail: On-chain attestations for every approval
Discord bots protect servers. Validify protects users‚Äîfrom announcement to transaction.
What is the Validify browser extension?

The Validify browser extension is free protection for Web3 users. When you visit a link that claims to be from a Web3 project, the extension checks:

  • Was this link approved through Validify's multi-sig verification?
  • Does the smart contract match what the project registered?
  • Are there any known threat indicators?

You'll see a clear trust indicator before you connect your wallet:
